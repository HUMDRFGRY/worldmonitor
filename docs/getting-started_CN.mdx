本指南涵盖在本地设置 WorldMonitor 所需的一切内容，从前置条件和安装，到理解项目结构和 API 依赖。

## 技术栈

| 层 | 技术 | 用途 |
|-------|------------|---------|
| **语言** | TypeScript 5.x | 覆盖 60+ 源文件的类型安全 |
| **构建** | Vite | 快速 HMR，优化后的生产构建 |
| **地图（桌面端）** | deck.gl + MapLibre GL | 面向大数据集的 WebGL 加速渲染 |
| **地图（移动端）** | D3.js + TopoJSON | 省电的 SVG 回退方案 |
| **并发** | Web Workers | 线程外聚类与关联分析 |
| **AI/ML** | ONNX Runtime Web | 浏览器端推理，用于离线摘要 |
| **网络** | WebSocket + REST | 实时 AIS 流，以及其他 API 的 HTTP 请求 |
| **存储** | IndexedDB | 快照、基线（数 MB 状态） |
| **偏好设置** | LocalStorage | 用户设置、监视器、面板顺序 |
| **部署** | Vercel Edge | 带全球分发的无服务器代理 |

### 地图渲染架构

地图采用针对不同平台优化的混合渲染策略：

**桌面端（deck.gl + MapLibre GL）**：

- WebGL 加速图层可以流畅处理数千个标记
- MapLibre GL 提供基础地图瓦片（OpenStreetMap）
- 针对不同数据类型使用 GeoJSON、Scatterplot、Path 和 Icon 图层
- 基于 GPU 的聚类和拾取实现响应式交互

**移动端（D3.js + TopoJSON）**：

- 使用 SVG 渲染以提升电池效率
- 减少标记数量并简化图层
- 通过更大的命中区域优化触控交互
- 当 WebGL 不可用时自动回退

### 关键库

- **deck.gl**：高性能 WebGL 可视化图层
- **MapLibre GL**：开源地图渲染引擎
- **D3.js**：SVG 地图渲染、缩放行为（移动端回退）
- **TopoJSON**：高效的地理数据编码
- **ONNX Runtime**：浏览器端机器学习推理
- **自定义 HTML 转义**：XSS 防护（DOMPurify 模式）

### 不使用外部 UI 框架

整个 UI 都是手写 DOM 操作，不使用 React、Vue 或 Angular。这样可以保持 bundle 较小（gzip 后约 250KB），并对渲染性能提供更细粒度的控制。

### 构建时配置

Vite 会在构建时注入配置值，从而实现自动版本同步等功能：

| 变量 | 来源 | 用途 |
|----------|--------|---------|
| `__APP_VERSION__` | `package.json` 的版本字段 | 头部显示当前版本 |

这确保显示的版本始终与发布包一致，无需手动同步。

```typescript
// vite.config.ts
define: {
  __APP_VERSION__: JSON.stringify(pkg.version),
}

// App.ts
const header = `World Monitor v${__APP_VERSION__}`;
```

## 安装

**要求：** Go 1.21+ 和 Node.js 18+。

```bash
# 克隆仓库
git clone https://github.com/koala73/worldmonitor.git
cd worldmonitor

# 安装全部内容（buf、sebuf 插件、npm 依赖、proto 依赖）
make install

# 启动开发服务器
npm run dev

# 生产构建
npm run build
```

如果你修改了任何 `.proto` 文件，请在构建或推送前重新生成：

```bash
make generate   # 重新生成 TypeScript 客户端、服务端和 OpenAPI 文档
```

完整 proto 工作流请参见 [添加端点](./adding-endpoints_CN.mdx)。

### 可选 API 密钥

某些功能需要 API 凭证。没有这些密钥时，对应图层会被隐藏：

| 变量 | 服务 | 获取方式 |
|----------|---------|------------|
| `FINNHUB_API_KEY` | 股票报价（主） | 在 [finnhub.io](https://finnhub.io/) 免费注册 |
| `EIA_API_KEY` | 石油分析 | 在 [eia.gov/opendata](https://www.eia.gov/opendata/) 免费注册 |
| `VITE_WS_RELAY_URL` | AIS 船舶追踪 | 部署 AIS relay 或使用托管服务 |
| `VITE_OPENSKY_RELAY_URL` | 军机追踪 | 使用 OpenSky 凭证部署 relay |
| `OPENSKY_CLIENT_ID` | OpenSky 认证（relay） | 在 [opensky-network.org](https://opensky-network.org) 免费注册 |
| `OPENSKY_CLIENT_SECRET` | OpenSky 认证（relay） | OpenSky 账户设置中的 API key |
| `CLOUDFLARE_API_TOKEN` | 互联网中断 | 带 Radar 访问权限的免费 Cloudflare 账户 |
| `ACLED_ACCESS_TOKEN` | 抗议数据（服务端） | 在 acleddata.com 免费注册 |
| `WINGBITS_API_KEY` | 飞机补充信息 | 联系 [Wingbits](https://wingbits.com) 获取 API 访问权限 |

没有这些密钥，仪表盘仍然可以完整运行。受影响的图层只是不显示。核心功能（新闻、市场、地震、天气）不需要任何配置。
