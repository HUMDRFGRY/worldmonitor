World Monitor 提供了一整套用于实时全球情报监控的功能，从交互式地图和数据图层，到市场追踪和可自定义的面板布局，一应俱全。

## 交互式全球地图

- **缩放与平移** - 使用鼠标/触控板手势进行平滑导航
- **区域聚焦** - 8 个预设视图可快速跳转（Global、Americas、Europe、MENA、Asia、Latin America、Africa、Oceania）
- **图层系统** - 可切换 20+ 个按类别组织的数据图层
- **时间过滤** - 按时间范围筛选事件（1h、6h、24h、48h、7d）
- **可固定地图** - 滚动面板时可将地图固定在顶部，或让它随页面滚动
- **智能标记聚类** - 低缩放级别时邻近标记会分组，放大时展开

## 标记聚类

密集区域中拥有大量数据点时，会使用智能聚类来避免视觉拥挤：

**工作方式**

- 在像素半径内的标记（会随缩放级别自适应）会合并为聚类徽标
- 聚类徽标显示被分组项目的数量
- 点击聚类会打开一个弹窗，列出所有被分组项目
- 放大后聚类半径会缩小，最终显示单个标记

**分组逻辑**

- **Protests**：仅在同一国家内聚类（骚乱优先排序，高严重度优先）
- **Tech HQs**：仅在同一城市内聚类（Big Tech 优先于独角兽，再优先于上市公司）
- **Tech Events**：仅在同一地点内聚类（按日期排序，最早的优先）

这可以防止像迪拜和利雅得的公司在全球缩放级别下被合并，同时在洲际尺度上仍保持清晰的可视化。

## 来源过滤

头部的 **SOURCES** 按钮会打开一个全局来源管理弹窗，让你对仪表盘里显示的新闻来源进行细粒度控制。

**能力：**

- **搜索**：按名称过滤来源列表，快速找到目标媒体
- **单个开关**：点击任意来源可启用/禁用
- **批量操作**：支持 "Select All" 和 "Select None"
- **计数显示**：显示 "45/77 enabled" 之类的状态
- **持久化**：设置保存到 localStorage，并跨会话保留

**使用场景：**

- **减少噪音**：禁用高流量聚合源（Google News），聚焦主流媒体
- **区域聚焦**：只启用特定地理区域相关的来源
- **来源质量**：禁用信噪比较差的来源
- **偏差管理**：通过启用/禁用具有特定编辑倾向的来源来平衡报道

**技术细节：**

- 禁用来源会在抓取时过滤，而不是在展示时过滤，从而减少带宽和 API 调用
- 会同时影响所有新闻面板：禁用一次 BBC，就会在所有地方消失
- 所有来源都被禁用时，面板会显示 "All sources disabled" 提示
- 变更会在下一次刷新周期生效

## 实时新闻流

嵌入来自主要新闻网络的 YouTube 直播流，并支持频道切换：

| 频道 | 覆盖 |
|---------|----------|
| **Bloomberg** | 商业与财经新闻 |
| **Sky News** | 英国与国际新闻 |
| **Euronews** | 欧洲视角 |
| **DW News** | 德国国际广播 |
| **France 24** | 法国全球新闻 |
| **Al Arabiya** | 中东新闻（阿语视角） |
| **Al Jazeera** | 中东与国际新闻 |

**核心功能：**

- **频道切换器** - 一键在各网络之间切换
- **直播指示器** - 闪烁点显示流状态，点击可暂停/播放
- **静音切换** - 音频控制（默认静音）
- **双宽面板** - 更大的视频播放器，便于观看

**性能优化：**

直播流面板使用 **YouTube IFrame Player API**，而不是原始 iframe 嵌入。这提供了几个优势：

| 功能 | 好处 |
|---------|---------|
| **持久播放器** | 静音/播放/频道切换时不会重新加载 iframe |
| **API 控制** | 可直接调用 `playVideo()`、`pauseVideo()`、`mute()` |
| **更少带宽** | 状态变化时继续播放同一流 |
| **更快切换** | 频道切换通过 `loadVideoById()` 完成 |

**空闲检测：**

为了节省资源，面板实现了自动空闲暂停：

| 触发条件 | 动作 |
|---------|--------|
| **标签页隐藏** | 流暂停（通过 Visibility API） |
| **5 分钟无操作** | 流暂停（无鼠标/键盘活动） |
| **用户返回** | 自动恢复播放 |
| **手动暂停** | 单独追踪用户意图 |

这可以防止后台标签页消耗带宽，同时保留用户对手动暂停流的偏好。
